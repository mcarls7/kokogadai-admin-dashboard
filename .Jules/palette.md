## 2026-05-30 - Contribution UX in Boilerplate Repositories
**Learning:** In projects lacking an interface (boilerplate state), the documentation (README) and contribution workflow (PR templates) serve as the primary user interface. Scannability via anchors and emoji-prefixing headers significantly reduces developer friction.
**Action:** Prioritize navigation anchors and 'Back to top' links in early-stage documentation. Use concise, high-density checklists in PR templates to maintain focus on UX without bloating the process.
